hexo框架 + github action自动化部署

准备工作

首先安装nodejs git 这里不说了自己网上找教程 nodejs 18版本

  1. Git是一个免费的、开源的版本控制软件。在Windows上安装git，一般为msysgit，
    官方下载地址 :http://code.google.com/p/msysgit/downloads/list
    安装步骤下一步即可…
  2. nodejs官网：https://nodejs.org/en/

判断是否安好

安装hexo

如上可以看到已经初步看到页面

把marster分支改成main

使用pure主题 https://github.com/cofess/hexo-theme-pure

根目录下新建__config.pure.yml 配置文件 将原有主题内容复制过去_

同理将pure下的__source中的内容复制到根目录的source里_,并新建一个image文件夹 放两张图片 一个是图片 一个是网站icon图标

主题配置文件下

favicon改了 avatar改了

站点配置文件下 language改了 theme改成pure

安装一系列需要的插件

页面大概如下

部署到github上

参考这篇文章 https://xulianjun.github.io/2024/09/08/Hexo%20+%20GitHub%20Action%EF%BC%8C%E9%9B%B6%E6%88%90%E6%9C%AC%E8%87%AA%E5%8A%A8%E5%8C%96%E9%83%A8%E7%BD%B2%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2/

  1. 创建仓库

建一个私有仓库

再建一个公有仓库

  1. 创建token

    点击右上角头像 ——> settings -> developer settings ->

创建token

token添加到公有仓库

下一步

注意用户名要小写

然后配置工作流

在根目录下的.github文件夹下新建workflows文件下新建deploy.yml文件

代码上传到源码仓库等待工作流安装完成注意本地为main分支

在展示仓库里setting>page里

等待3 5分钟网站出现

网站出现后没有样式

  1. 用户名没有小写

  2. config.yml里配置root和url

    重新push提交看效果

搭建完成
